# AI大模型 API 测试指南

## 测试目标

验证所有 AI 功能已成功切换到AI大模型，并正确处理 Markdown 格式。

## 测试 1：算命功能（generate_fortune_detail）

### 步骤

1. **打开开发者工具**
   - 按 F12
   - 切换到 Console 标签

2. **进行测算**
   - 进入"开始测算"页面
   - 输入测试数据：
     ```
     姓名：李四
     性别：女
     出生日期：1995-08-20
     出生时间：10:30
     近期事件：最近感情不顺
     关注领域：婚姻
     当前心情：迷茫
     ```
   - 点击"开始测算"

3. **查看日志**

**✅ 成功标志**：
```
🔮 开始调用 AI 生成详细运势...
🚀 使用原生 fetch 调用 Edge Function...
✅ Edge Function 请求完成: 200
🤖 开始调用AI大模型 API...
📡 AI大模型 API 响应状态: 200
✅ AI大模型 API 返回成功
🧹 清理后内容长度: ...
✅ JSON 解析成功
✅ AI 生成运势成功！
```

**关键点**：
- ✅ 日志显示"AI大模型 API"（不是 MiniMax）
- ✅ 有"清理后内容长度"日志（说明 Markdown 清理生效）
- ✅ JSON 解析成功

4. **验证内容**

在结果页面，查看内容：
- **内容长度**：每个维度 300-600 字，总计 2500+ 字
- **格式检查**：
  - ❌ 不应该有 `**粗体**`
  - ❌ 不应该有 `## 标题`
  - ❌ 不应该有 `- 列表`
  - ❌ 不应该有 `> 引用`
  - ✅ 应该是纯文本，段落清晰

**示例（正确）**：
```
李四女士，您生于1995年8月20日10:30，八字为【乙亥 甲申 丙寅 癸巳】。

从您的八字来看，命中木旺，木主仁慈、温和、善良。您性格温柔体贴，善解人意，但有时也会过于优柔寡断。

近期您提到感情不顺，这与您当前的运势有关。2026年2月正值春季，木旺之时，您的感情运势会有所波动...
```

**示例（错误，有 Markdown）**：
```
**李四女士**，您生于1995年8月20日10:30，八字为【乙亥 甲申 丙寅 癸巳】。

## 八字分析

从您的八字来看：
- 命中木旺
- 木主仁慈、温和、善良
- 性格温柔体贴

> 近期您提到感情不顺...
```

```
📡 AI大模型 API 响应状态: 200
📝 开始读取流式响应...
```

6. **验证回答**

**预期效果**：
- ✅ 回答逐字显示（流式）
- ✅ 内容详实（500-1000 字）
- ✅ 结合八字信息（提到"乙亥 甲申 丙寅 癸巳"）

- ✅ 格式正确（纯文本，没有 Markdown 符号）

**示例（正确）**：
```
李四女士，从您的八字【乙亥 甲申 丙寅 癸巳】来看，您命中木旺，性格温柔善良，但有时也会过于敏感。

您提到最近和对象总是吵架，这与您当前的运势有关。2026年2月正值春季，木旺之时，您的情绪会比较敏感，容易因为小事而产生矛盾。

从命理角度来看，您和对象是否合适，需要看双方的八字是否相合。但从您的描述来看，问题可能不在于是否合适，而在于沟通方式。

建议您：
1. 时间选择：在农历三月（阳历4月）或农历八月（阳历9月）进行深入沟通，这两个时间段您的运势较好，更容易达成共识。
2. 方位建议：在家中的东方或东南方进行沟通，这两个方位对您有利。
3. 沟通技巧：尝试用更温和的方式表达您的想法，避免情绪化的争吵。

至于是否分手，建议您不要在情绪激动时做决定。给彼此一些时间和空间，冷静下来后再做判断。
```

## 测试 3：Markdown 处理验证

### 目的

验证即使 AI 返回 Markdown 格式，系统也能正确处理。

### 方法

如果在测试 1 或测试 2 中发现 AI 返回了 Markdown 格式：

1. **查看原始内容**（在控制台）
2. **查看显示内容**（在页面）
3. **对比差异**

**预期**：
- 原始内容可能有 Markdown 符号
- 显示内容应该是纯文本或正确的 HTML

**示例**：

**原始内容**（控制台）：
```
**标题**

这是一段文字。

- 列表项1
- 列表项2
```

**显示内容**（页面）：
```
标题

这是一段文字。

列表项1
列表项2
```

或者（如果使用 markdownToHtml）：
```html
<strong>标题</strong>

这是一段文字。

列表项1
列表项2
```

## 测试 4：对比测试

### 目的

验证AI大模型的输出质量。

### 步骤

**测试 A：相同输入，多次测算**
1. 使用完全相同的信息，测算 2 次
2. 对比结果：
   - 八字、五行、幸运色应相同
   - 详细分析内容应有差异（AI 随机性）

**测试 B：修改近期事件**
1. 第一次：近期事件 = "工作压力大"
2. 第二次：近期事件 = "感情不顺"
3. 对比结果：
   - 事业运势应特别关注工作问题（第一次）
   - 婚姻感情应特别关注感情问题（第二次）

**测试 C：咨询不同问题**
1. 第一次：问工作相关问题
2. 第二次：问感情相关问题
3. 对比结果：
   - 回答应针对不同问题
   - 建议应有针对性

## 常见问题

### Q1: 日志显示"MiniMax API"而不是"AI大模型 API"

**原因**：Edge Function 没有重新部署

**解决**：
1. 确认 Edge Function 已重新部署
2. 清除浏览器缓存
3. 重新测试

### Q2: 内容中仍然有 Markdown 符号

**原因**：
1. AI 没有遵守系统提示词
2. Markdown 清理函数没有生效

**解决**：
1. 查看控制台日志，确认是否有"清理后内容长度"
2. 如果没有，说明清理函数没有执行
3. 检查 Edge Function 代码

### Q3: 咨询功能没有响应

**原因**：
1. 字段访问错误（input vs input_data）
2. Edge Function 错误

**解决**：
1. 查看控制台日志
2. 查看 Edge Function 日志
3. 确认字段名正确

### Q4: 流式显示不流畅

**原因**：网络延迟或 API 响应慢

**解决**：等待网络恢复，重试
