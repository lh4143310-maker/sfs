# AI 个性化增强优化说明

## 问题描述

用户反馈：不同的人和八字，但 AI 生成的运势建议出现大量重复内容。

**具体问题**：
1. **时间建议重复**：不同的人，事业运势都建议"5月和8月"
2. **内容模板化**：财运、健康、感情等维度的内容也有大量重复
3. **个性化不足**：没有根据具体的八字、五行分布给出差异化的建议

## 问题分析

### 1. AI大模型 API 的限制

**与 MiniMax 的区别**：
- MiniMax 有 `temperature` 参数（0-1），可以控制随机性
- AI大模型没有 `temperature` 参数，随机性由模型内部控制
- AI大模型更倾向于生成稳定、一致的内容

### 2. 系统提示词不够强

**原有提示词的问题**：
- 只是泛泛地说"每次分析都要有独特见解"
- 没有明确禁止使用模板化内容
- 没有给出具体的个性化要求
- 没有根据五行属性计算适合的月份

### 3. 缺少随机种子

**问题**：
- 相同的输入，AI 倾向于生成相似的输出
- 没有引入随机元素来增加差异性

## 解决方案

### 1. 根据五行计算适合的月份

**核心逻辑**：
```typescript
const calculateLuckyMonths = () => {
  const elementMonths: Record<string, number[]> = {
    metal: [7, 8, 9],      // 金旺于秋
    wood: [1, 2, 3],       // 木旺于春
    water: [10, 11, 12],   // 水旺于冬
    fire: [4, 5, 6],       // 火旺于夏
    earth: [3, 6, 9, 12]   // 土旺于四季末月
  };
  
  // 根据主五行和弱五行计算
  const strongMonths = elementMonths[dominantElement[0]] || [];
  const weakMonths = elementMonths[weakElement[0]] || [];
  
  // 避开弱五行的月份，选择主五行的月份
  const luckyMonths = strongMonths.filter(m => !weakMonths.includes(m));
  
  return luckyMonths.length > 0 ? luckyMonths : strongMonths;
};
```

**效果**：
- 五行金旺的人：适合 7、8、9 月（秋季）
- 五行木旺的人：适合 1、2、3 月（春季）
- 五行水旺的人：适合 10、11、12 月（冬季）
- 五行火旺的人：适合 4、5、6 月（夏季）
- 五行土旺的人：适合 3、6、9、12 月（四季末月）

**个性化**：
- 每个人的五行分布不同
- 计算出的适合月份也不同
- 不会再出现"都是5月和8月"的问题

## 五行与月份对应关系

### 传统命理依据

**五行旺季**：
- **金**：旺于秋季（申酉戌月，对应 7、8、9 月）
- **木**：旺于春季（寅卯辰月，对应 1、2、3 月）
- **水**：旺于冬季（亥子丑月，对应 10、11、12 月）
- **火**：旺于夏季（巳午未月，对应 4、5、6 月）
- **土**：旺于四季末月（辰未戌丑月，对应 3、6、9、12 月）

**生克关系**：
- 金克木：金旺时（秋季），木弱
- 木克土：木旺时（春季），土弱
- 土克水：土旺时（四季末月），水弱
- 水克火：水旺时（冬季），火弱
- 火克金：火旺时（夏季），金弱

**应用**：
- 五行旺的人，在其旺季运势最佳
- 五行弱的人，应避开克制其的季节

## 技术实现细节

### 1. 五行月份映射

**数据结构**：
```typescript
const elementMonths: Record<string, number[]> = {
  metal: [7, 8, 9],      // 金旺于秋
  wood: [1, 2, 3],       // 木旺于春
  water: [10, 11, 12],   // 水旺于冬
  fire: [4, 5, 6],       // 火旺于夏
  earth: [3, 6, 9, 12]   // 土旺于四季末月
};
```

### 2. 计算逻辑

**步骤**：
1. 获取主五行对应的月份
2. 获取弱五行对应的月份
3. 过滤掉弱五行的月份（避开不利时期）
4. 返回适合的月份

**代码**：
```typescript
const strongMonths = elementMonths[dominantElement[0]] || [];
const weakMonths = elementMonths[weakElement[0]] || [];
const luckyMonths = strongMonths.filter(m => !weakMonths.includes(m));
return luckyMonths.length > 0 ? luckyMonths : strongMonths;
```

### 3. 格式化输出

**代码**：
```typescript
const luckyMonthsStr = luckyMonths.map(m => `${m}月`).join('、');
// 输出：7月、8月、9月
```

### 4. 在 Prompt 中使用

**系统提示词**：
```
- 根据五行分析，此人适合的月份是：${luckyMonthsStr}
- 请在分析中使用这些月份，而不是固定的5月、8月等
```

**用户提示词**：
```
- 根据五行属性，此人适合的月份是：${luckyMonthsStr}（请在分析中使用这些月份）
- 结合当前时间（${currentYear}年${currentMonth}月）和适合月份（${luckyMonthsStr}），预测未来3-6个月的事业走势
```
