## 常见错误及解决方案

### 错误 1：环境变量未配置

**日志特征**：
```
❌ [微信登录] 环境变量未配置
❌ [微信登录] APP_ID: 未配置
❌ [微信登录] APP_SECRET: 未配置
```

**错误信息**：
```
微信配置缺失，请联系管理员配置 WECHAT_MINIPROGRAM_LOGIN_APP_ID 和 WECHAT_MINIPROGRAM_LOGIN_APP_SECRET
```

**解决方案**：
1. 登录 Supabase Dashboard
2. 进入 "Settings" → "Edge Functions" → "Secrets"
3. 添加以下两个环境变量：
   - `WECHAT_MINIPROGRAM_LOGIN_APP_ID`
   - `WECHAT_MINIPROGRAM_LOGIN_APP_SECRET`
4. 保存后等待几秒钟让配置生效

---

### 错误 2：微信登录凭证无效（40029）

**日志特征**：
```
❌ [微信登录] 微信接口返回错误
❌ [微信登录] 错误码: 40029
❌ [微信登录] 错误信息: invalid code
```

**错误信息**：
```
微信接口错误 (40029): 微信登录凭证无效，请重试
```

**原因**：
- 微信登录 code 已过期（5分钟有效期）
- 微信登录 code 已被使用（只能使用一次）
- AppID 或 AppSecret 配置错误

**解决方案**：
1. 重新点击"微信一键登录"按钮
2. 检查 AppID 和 AppSecret 是否正确
3. 确保没有重复调用登录接口

---

### 错误 3：微信登录频率限制（45011）

**日志特征**：
```
❌ [微信登录] 微信接口返回错误
❌ [微信登录] 错误码: 45011
❌ [微信登录] 错误信息: api minute-quota reach limit
```

**错误信息**：
```
微信接口错误 (45011): 微信登录频率限制，请稍后再试
```

**原因**：
- 短时间内调用次数过多
- 微信接口有频率限制

**解决方案**：
- 等待 1-2 分钟后再试
- 避免频繁点击登录按钮

---

### 错误 4：微信登录凭证已过期（40163）

**日志特征**：
```
❌ [微信登录] 微信接口返回错误
❌ [微信登录] 错误码: 40163
❌ [微信登录] 错误信息: code been used
```

**错误信息**：
```
微信接口错误 (40163): 微信登录凭证已过期，请重新登录
```

**原因**：
- 同一个 code 被使用了多次

**解决方案**：
- 重新点击"微信一键登录"按钮获取新的 code

---

### 错误 5：Token 验证失败

**日志特征**：
```
✅ [微信登录] 获取到 token: abc123...
🔐 [微信登录] 步骤5: 验证 token
❌ [微信登录] Token 验证错误: {...}
```

**错误信息**：
```
登录验证失败: Token has expired or is invalid
```

**原因**：
- Magic Link token 已过期
- Token 格式错误
- Supabase 认证配置问题

**解决方案**：
1. 检查 Supabase 认证配置
2. 确保 Edge Function 正确生成 token
3. 重新尝试登录

## 常见问题 FAQ

### Q1: 为什么在浏览器中无法使用微信登录？

**A**: 微信登录只能在微信小程序环境中使用。在浏览器中会显示提示信息：
```
微信登录仅在微信小程序中可用
网页端请使用用户名密码登录
```

---

### Q2: 为什么点击登录按钮没有反应？

**A**: 可能的原因：
1. 未勾选"我已阅读并同意《用户协议》和《隐私政策》"
2. 网络连接问题
3. 环境变量未配置

**解决方案**：
1. 确保勾选协议复选框
2. 检查网络连接
3. 查看控制台日志

---

### Q3: 为什么显示"微信登录失败"但没有详细信息？

**A**: 这通常是因为：
1. Edge Function 返回的错误格式不正确
2. 网络请求失败
3. 前端解析错误信息失败

**解决方案**：
1. 查看控制台的详细日志
2. 查看 Edge Function 日志
3. 检查网络请求的响应

---

### Q4: 如何验证环境变量是否配置正确？

**A**: 查看 Edge Function 日志：
```
📦 [微信登录] 环境变量状态: {
  hasAppId: true,
  hasAppSecret: true,
  appIdLength: 18,
  appSecretLength: 32
}
```

如果显示 `false` 或长度为 0，说明未配置。

---

### Q5: 为什么第一次登录成功，第二次就失败了？

**A**: 可能的原因：
1. 微信登录 code 只能使用一次
2. 频率限制

**解决方案**：
- 每次登录都会获取新的 code，不应该出现这个问题
- 如果出现，等待几秒钟后重试
