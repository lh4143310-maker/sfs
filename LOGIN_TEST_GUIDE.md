# 登录功能测试指南

## 快速测试步骤

### 1. 注册新用户

**操作步骤**：
1. 打开小程序，进入登录页面
2. 点击"没有账号？立即注册 →"切换到注册模式
3. 输入用户名：`test001`（3-20个字符，只能包含字母、数字、下划线）
4. 输入密码：`123456`（至少6个字符）
5. 勾选"我已阅读并同意《用户协议》和《隐私政策》"
6. 点击"注册账户"按钮

**预期结果**：
- ✅ 显示"注册成功"提示
- ✅ 1.5秒后自动跳转到首页
- ✅ 可以正常使用小程序功能

**可能的错误**：
- ❌ "请先阅读并同意用户协议和隐私政策" → 需要勾选协议
- ❌ "用户名至少3个字符" → 用户名太短
- ❌ "密码至少6个字符" → 密码太短
- ❌ "用户名只能包含字母、数字和下划线" → 用户名格式错误

### 2. 登录已有用户

**操作步骤**：
1. 如果已注册，在登录页面确保处于"登录"模式
2. 输入用户名：`test001`
3. 输入密码：`123456`
4. 勾选协议
5. 点击"立即登录"按钮

**预期结果**：
- ✅ 显示"登录成功"提示
- ✅ 跳转到首页或之前访问的页面

**可能的错误**：
- ❌ "用户名或密码错误" → 凭证不匹配，请检查输入

### 3. 微信一键登录（仅小程序环境）

**操作步骤**：
1. 在微信小程序中打开登录页面
2. 勾选协议
3. 点击"微信一键登录"按钮（绿色按钮）

**预期结果**：
- ✅ 显示"微信登录成功"提示
- ✅ 自动创建或关联账号
- ✅ 跳转到首页

**网页环境**：
- 显示提示信息："微信登录仅在微信小程序中可用"
- 引导使用用户名密码登录

### 4. 测试路由守卫

**操作步骤**：
1. 未登录状态下，尝试访问受保护页面（如"记录"或"我的"）
2. 应该自动跳转到登录页
3. 登录成功后，自动返回之前想访问的页面

**预期结果**：
- ✅ 未登录时无法访问受保护页面
- ✅ 登录后自动跳转回原页面

### 5. 测试退出登录

**操作步骤**：
1. 登录后，进入"我的"页面
2. 点击"退出登录"按钮
3. 确认退出

**预期结果**：
- ✅ 显示"已退出登录"提示
- ✅ 跳转到首页
- ✅ 再次访问受保护页面时需要重新登录

## 常见问题排查

### 问题1：点击注册/登录按钮没有反应

**可能原因**：
1. 未勾选隐私协议
2. 用户名或密码为空
3. 网络连接问题

**解决方法**：
1. 确保勾选协议复选框
2. 检查输入框是否有内容
3. 检查网络连接
4. 查看控制台错误日志

### 问题2：注册成功但无法登录

**可能原因**：
1. 数据库触发器未正确执行
2. Profile 记录未创建

**解决方法**：
1. 检查 Supabase 数据库中的 `auth.users` 表
2. 检查 `public.profiles` 表是否有对应记录
3. 查看触发器日志

### 问题3：微信登录失败

**可能原因**：
1. 不在微信小程序环境
2. Edge Function 未部署或配置错误
3. 微信 AppID/AppSecret 未配置

**解决方法**：
1. 确认在微信开发者工具或真机微信中运行
2. 检查 Edge Function `wechat_miniapp_login` 是否已部署
3. 检查环境变量：
   - `WECHAT_MINIPROGRAM_LOGIN_APP_ID`
   - `WECHAT_MINIPROGRAM_LOGIN_APP_SECRET`

### 问题4：登录后立即退出

**可能原因**：
1. Session 未正确保存
2. AuthContext 状态管理问题

**解决方法**：
1. 检查浏览器/小程序存储权限
2. 查看 AuthContext 的 session 状态
3. 检查 Supabase 客户端配置

## 调试技巧

### 1. 查看控制台日志

登录过程中会输出详细日志：
```
开始微信登录...
微信登录 code: xxxxx
获取到 token，开始验证
微信登录成功
```

### 2. 检查网络请求

在开发者工具的网络面板中查看：
- Supabase Auth API 请求
- Edge Function 调用
- 响应状态码和错误信息

### 3. 数据库查询

使用 Supabase Dashboard 或 SQL 查询：
```sql
-- 查看所有用户
SELECT id, email, created_at, confirmed_at 
FROM auth.users 
ORDER BY created_at DESC;

-- 查看所有 profiles
SELECT id, nickname, role, created_at 
FROM public.profiles 
ORDER BY created_at DESC;
```

### 4. 清除缓存

如果遇到奇怪的问题，尝试：
1. 清除小程序缓存
2. 清除浏览器缓存
3. 重新编译项目
