# AI 增强功能验证指南

## 快速验证步骤

### 步骤 1：打开浏览器开发者工具

1. 在浏览器中打开小程序
2. 按 F12 打开开发者工具
3. 切换到 Console（控制台）标签

### 步骤 2：进行一次测算

1. 进入"开始测算"页面
2. 输入以下测试数据：
   ```
   姓名：张三
   性别：男
   出生日期：1990-05-15
   出生时间：14:30
   近期事件：最近工作压力很大
   关注领域：事业
   当前心情：焦虑不安
   ```
3. 点击"开始测算"按钮
4. 等待 5-15 秒

在结果页面，查看任意一个维度的内容长度：

### 步骤 5：验证随机性

使用完全相同的信息，测算两次：

**预期结果**：
- 八字相同：庚午 辛巳 甲辰 辛未
- 五行相同：金45% 火20% 木15% 土20% 水0%
- 幸运色相同：黑色、蓝色、灰色
- 幸运数字相同：1、6、4
- **但详细分析内容有差异**（AI 的随机性）

如果两次内容完全一样，说明使用了本地算法。

## 常见问题排查

### 问题 1：看到"AI 生成失败，使用本地算法"

**可能原因**：
1. 网络问题
2. Edge Function 未正确部署
3. API Key 未配置
4. API 限额用尽

**排查步骤**：
1. 检查网络连接
2. 查看 Edge Function 日志（Supabase Dashboard → Functions → generate_fortune_detail → Logs）
3. 检查环境变量 INTEGRATIONS_API_KEY 是否配置
4. 查看 MiniMax API 使用情况

### 问题 2：请求超时

**症状**：
```
AbortError: signal is aborted without reason
FunctionsFetchError: Failed to send a request to the Edge Function
```

**解决方案**：
- 已修复：增加了 Taro.request 超时时间到 60 秒
- 如果仍然超时，可能是网络问题或 API 响应太慢

### 问题 3：内容长度不够

**症状**：
- 每个维度只有 100-200 字
- 内容比较笼统

**原因**：
- 使用了本地算法（AI 调用失败）

**解决方案**：
- 查看控制台日志，确认错误原因
- 根据错误信息进行排查

### 问题 4：内容千篇一律

**症状**：
- 多次测算，内容完全一样
- 修改输入，内容变化不大

**原因**：
- 使用了本地算法（AI 调用失败）

**解决方案**：
- 确认 AI 是否正常工作（查看日志）
- 如果 AI 正常工作，每次生成的内容都会有差异
