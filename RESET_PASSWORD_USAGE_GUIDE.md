### 步骤1：访问页面
1. 打开浏览器
2. 启动开发服务器：`npm run dev:h5`
3. 访问：`http://localhost:10086/#/pages/reset-password/index`

### 步骤2：查看调试区域
页面顶部应该显示一个**黄色背景**的调试区域：

```

│ 调试信息（测试用）                    │
```

2. 观察页面变化

- 调试信息显示"当前步骤: reset"
- 下方显示重置密码表单
- 包含两个密码输入框
- 包含"确认重置密码"按钮

1. 在"新密码"框输入：`test123456`

## 常见问题解答

### Q1：页面完全空白，什么都没有
**A**：这是最严重的问题，可能原因：
1. 路由配置错误 - 检查 `app.config.ts`
2. 文件不存在 - 检查 `src/pages/reset-password/index.tsx`
3. 编译错误 - 运行 `npm run lint`
4. 缓存问题 - 清除缓存，重新编译

**解决步骤**：
```bash
# 1. 检查文件
ls -la src/pages/reset-password/

# 2. 检查路由
grep "reset-password" src/app.config.ts

# 3. 运行 lint
npm run lint

# 4. 清除缓存重新编译
rm -rf .temp dist
npm run dev:h5
```

### Q2：页面显示但没有调试区域
**A**：可能是代码未更新或缓存问题

**解决步骤**：
1. 强制刷新浏览器（Ctrl+F5）
2. 清除浏览器缓存
3. 重新编译项目

### Q3：调试按钮不工作
**A**：可能是 JavaScript 错误

**解决步骤**：
1. 打开浏览器控制台
2. 查看是否有错误信息
3. 检查按钮的 onClick 事件

### Q4：切换到重置步骤后没有表单
**A**：这是条件渲染问题

**解决步骤**：
1. 确认调试信息显示"当前步骤: reset"
2. 检查代码中的 `{step === 'reset' && (...)}`
3. 在控制台执行：
   ```javascript
   document.querySelectorAll('[class*="reset"]')
   ```

### Q5：可以看到表单但无法输入
**A**：可能是 Input 组件问题

**解决步骤**：
1. 检查 Input 组件是否正确导入
2. 检查是否有 CSS 覆盖
3. 尝试点击其他输入框
