### 3. 手机号登录功能增强
**问题**：有手机号和验证码，但登录失败

**修复**：
1. 修改了 `phone-login` Edge Function，使用 `recovery` 类型生成令牌（更适合第三方验证后的登录）
2. 在 `AuthContext.tsx` 中添加了详细的日志输出
3. 修改了 `verifyOtp` 调用，使用 `recovery` 类型

**测试方法**：
1. 打开浏览器控制台（F12）
2. 打开登录页面
3. 切换到"手机登录"
4. 输入手机号：18600737571
5. 点击"获取验证码"
6. 输入收到的验证码
7. 点击"立即登录"
8. 查看控制台输出

**控制台日志说明**：
```
开始手机号登录，手机号: 18600737571 sessionId: xxx
phone-login 返回结果: { data: {...}, funcError: null }
获取到 token_hash、email 和 type: { token_hash: 'xxx', email: 'xxx', type: 'recovery' }
开始调用 verifyOtp，type: recovery
verifyOtp 返回结果: { authData: {...}, authError: null }
手机号登录成功
```

**可能的错误情况**：

#### 错误1：验证码错误
```
phone-login 返回失败: { error: '验证码错误' }
```
**解决方案**：重新获取验证码

#### 错误2：手机号未注册
```
phone-login 返回失败: { error: '该手机号未注册，请先注册' }
```
**解决方案**：先注册账号

#### 错误3：verifyOtp 失败
```
verifyOtp 失败: { message: 'Token has expired or is invalid' }
```
**可能原因**：
- 令牌已过期
- 令牌格式不正确
- Edge Function 返回的 token_hash 有问题

**解决方案**：
- 检查 Edge Function 日志
- 确认 `generateLink` 返回的 token 是否正确

#### 错误4：未创建会话
```
未创建会话
```
**可能原因**：
- `verifyOtp` 成功但没有返回 session
- 用户账号状态异常

**预期结果**：✅ 登录成功，跳转到首页

### 4. 密码重置功能验证
**问题**：需要验证密码重置页面的功能是否可用

**功能说明**：
密码重置页面有三个步骤：
1. **邮箱步骤**：输入邮箱，发送重置链接
2. **验证步骤**：等待邮件，点击邮件中的链接
3. **重置步骤**：输入新密码，确认密码

**测试方法**：

#### 步骤1：发送重置邮件
1. 打开重置密码页面
2. 输入已注册的邮箱地址
3. 点击"发送重置链接"
4. 等待60秒倒计时
5. 检查邮箱是否收到重置邮件

**预期结果**：
- ✅ 显示"重置链接已发送到您的邮箱"
- ✅ 按钮显示60秒倒计时
- ✅ 倒计时数字为黑色，清晰可见
- ✅ 收到重置邮件

#### 步骤2：点击邮件链接
1. 打开邮箱
2. 找到重置密码邮件
3. 点击邮件中的重置链接
4. 页面应该自动跳转到重置步骤

**预期结果**：
- ✅ 页面显示"请设置新密码"提示
- ✅ 显示新密码和确认密码输入框
- ✅ 显示"重置密码"按钮

#### 步骤3：重置密码
1. 输入新密码（至少6个字符）
2. 输入确认密码（与新密码一致）
3. 点击"重置密码"按钮
4. 等待处理

**预期结果**：
- ✅ 显示"密码重置成功"
- ✅ 2秒后自动跳转到登录页面
- ✅ 可以使用新密码登录

**可能的错误情况**：

#### 错误1：会话失效
**错误信息**：授权会话失效，请重新点击邮件链接

**原因**：
- 邮件链接已过期（通常1小时）
- 页面停留时间过长
- 浏览器清除了 cookies

**解决方案**：
1. 返回重置密码页面
2. 重新发送重置邮件
3. 立即点击新邮件中的链接
4. 在5分钟内完成密码重置

#### 错误2：密码不一致
**错误信息**：两次输入的密码不一致

**解决方案**：确保两次输入的密码完全一致

#### 错误3：密码太短
**错误信息**：密码至少6个字符

**解决方案**：输入至少6个字符的密码

## 常见问题排查

### Q1：手机号登录一直失败
**排查步骤**：
1. 打开控制台查看详细日志
2. 确认手机号格式正确（11位，1开头）
3. 确认验证码正确（6位数字）
4. 确认验证码未过期（5分钟内）
5. 检查是否已注册该手机号

### Q2：验证码收不到
**可能原因**：
- 手机号输入错误
- 短信服务延迟
- 手机拦截了短信
- 短信配额用完

**解决方案**：
- 检查手机号是否正确
- 等待1-2分钟
- 检查垃圾短信箱
- 联系技术支持

### Q3：密码重置邮件收不到
**可能原因**：
- 邮箱地址错误
- 邮件被拦截
- 邮件服务延迟

**解决方案**：
- 检查邮箱地址是否正确
- 检查垃圾邮件箱
- 等待5分钟后重试
- 使用其他邮箱

### Q4：倒计时看不见
**已修复**：倒计时数字颜色已改为黑色

**如果仍然看不见**：
- 清除浏览器缓存
- 刷新页面
- 检查是否使用了自定义主题

## 技术细节

### 手机号登录流程
```
用户输入手机号和验证码
    ↓
前端调用 signInWithPhone
    ↓
调用 phone-login Edge Function
    ↓
Edge Function 验证验证码
    ↓
Edge Function 查找用户
    ↓
Edge Function 生成 recovery 令牌
    ↓
返回 token_hash 和 email
    ↓
前端调用 verifyOtp (type: recovery)
    ↓
创建登录会话
    ↓
登录成功
```

### 密码重置流程
```
用户输入邮箱
    ↓
调用 resetPasswordForEmail
    ↓
Supabase 发送重置邮件
    ↓
用户点击邮件链接
    ↓
Supabase 验证令牌
    ↓
创建临时会话
    ↓
页面检测到 PASSWORD_RECOVERY 事件
    ↓
切换到重置步骤
    ↓
用户输入新密码
    ↓
调用 updateUser
    ↓
密码重置成功
```
